==================================================
CUSTOMER QR MENU ORDERING SYSTEM
================================

CUSTOMER MENU SYSTEM

Each tenant must have its own public customer ordering page.

Customers can access menu via:

* QR Code
* Public Link
* Table QR
* WhatsApp Link

Example:

yourapp.com/store/coffee-jakarta
yourapp.com/menu/tenant-slug

Each tenant has isolated customer menu pages.

==================================================
CUSTOMER FEATURES
=================

Customers can:

* Browse menu
* View categories
* View product details
* Add to cart
* Add notes
* Select variants
* Select toppings/add-ons
* Change quantity

==================================================
ORDER TYPES
===========

Customer must choose:

* Dine In
* Take Away
* Delivery

==================================================
DINE IN FLOW
============

* Table Number
* Customer Name

==================================================
TAKE AWAY FLOW
==============

Customer inputs:

* Customer Name
* Phone Number

==================================================
DELIVERY FLOW
=============

Customer inputs:

* Customer Name
* Phone Number
* Full Address
* Delivery Notes
* Google Maps Location

Delivery system supports:

* Nearby Delivery
* Long Distance Delivery

Delivery fee can be:

* Manual
* Distance-based
* Fixed Price

==================================================
MULTI TENANT CUSTOMER MENU
==========================

Each tenant must have its own isolated public ordering system.

Example:

Tenant A:
Cafe Kopi

QR:
kopi.app/menu/kopijakarta

Tenant B:
Burger Store

QR:
kopi.app/menu/burgerbandung

All data must be automatically filtered by:

* tenant_id

Including:

* products
* orders
* payments
* customers
* delivery data

Each business owner can only view their own data and orders.
